Asset Simulator HTTP Server

¿Qué es este servidor?

Es un servidor HTTP para simular la realización de tareas por parte de diferentes activos. Sirve para validar el enfoque SMIA y para tareas de mantenimiento o simulación de activos industriales.

Referencia de la API

Documentación técnica para el control e integración externa de la flota de activos.

GET /api/v1/asset/{asset_id}/status

Recupera la telemetría actual de un activo específico. Devuelve el estado en tiempo real, el nivel de batería restante y la tarea que está ejecutando en caso de estar ocupado.

Parámetros de Ruta

  • asset_id (string) - ID único del activo (ej. ROBOT_A)

Respuesta (200 OK)

    {
      "type": "robot",
      "status": "idle",
      "battery": 100.0,
      "current_task": null
    }
POST /api/v1/asset/{asset_id}/action/{action_name}

Asigna una tarea a un activo. La acción a ejecutar se indica como segmento de la ruta. El servidor validará si la tarea es compatible con el tipo de activo, si está disponible (estado 'idle') y si tiene batería suficiente para completar la operación solicitada.

Parámetros de Ruta

  • asset_id (string) - ID único del activo.
  • action_name (string) - Tarea válida soportada por el tipo de activo.

Body Request (JSON)

  • duration (int) - Segundos (1 a 30).

Ejemplo cURL

curl -X POST http://localhost:5000/api/v1/asset/ROBOT_A/action/weld -H "Content-Type: application/json" -d '{"duration": 15}'

Respuesta (200 OK)

    {
      "status": "success",
      "asset": "ROBOT_A",
      "action_completed": "weld"
    }
POST /api/v1/asset/{asset_id}/charge

Alterna el estado de recarga de un activo. Para iniciar la recarga, el activo no puede estar realizando ninguna tarea. Si alcanza el 100%, el sistema lo devolverá automáticamente a estado 'idle'.

Body Request (JSON)

  • enable (boolean) - True para cargar, False para detener.

Respuesta (200 OK)

    {
      "status": "success",
      "charging": true
    }
